## 本次议题

### UTsweetyfish: GenAI/LLM/AI Agent Guideline

https://github.com/deepin-community/rfcs/issues/19

UTsweetyfish: 介绍了目前现状，概要内容与 issue 内容一致
BLumia: 1. 关于确定使用的名称（使用哪个term）的实际意义？ 2. 我们目前是否有涉及到关于声明方式的要求/约束？ 3, 关于可以实际实施的内容？
zccrs: 用户不需要做此类披露，我们要求所有项目的 review 仍然以 PR 质量本身为主。期望上以 commit message 格式规范形式。我们允许在例如PR描述中披露AI使用情况，但作为现阶段的commit 规范，commit msg 中不应披露，避免潜在的不明朗的政策影响。
blumia: 关于 commit msg 的约束，如果后续明确确认标注相关披露字段无影响，是否可以移除这个限制
zccrs: 可以
UTsweetyfish: CLA 是否需要调整？
zccrs: 这个需要法务确认。
Avenger-285714: 对于commit msg不允许出现 LLM 来源的约束建议明确标注是临时约束。
zccrs: 可以进行标注。

目前的结论：可以接受贡献者主动进行披露，不强制要求披露，但不应写在代码仓库，或 git commit 的 assisted-by/co-authered-by 字段以免潜在的风险。允许项目在源码目录放置 AGENTS.md 或同类性质的配置文件/描述。可以经过 RFC 形式详细制定相关的指导建议并实际展开决议。

Revy: 我不认可上述结论，对于AI留痕是允许开发者自行决定，如果不允许留痕属于掩耳盗铃行为，应该禁止AI贡献，至少要可允许标注assisted-by: yes 至于工具本身可以不标记名字，避免广告嫌疑
